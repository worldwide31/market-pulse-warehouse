# Методология двенадцати факторов

Проект `МаркетПульс` реализован с учетом методологии 12 факторов для облачных клиент-серверных приложений.

## 1. Кодовая база

Одна кодовая база хранится в Git и опубликована в GitHub. Из нее разворачиваются локальная, тестовая и облачная версии приложения.

Репозиторий: `https://github.com/worldwide31/market-pulse-warehouse`

## 2. Зависимости

Backend-зависимости явно перечислены в `backend/requirements.txt`. Docker-образ устанавливает зависимости командой `pip install -r requirements.txt`, поэтому окружение воспроизводимо.

Frontend не требует пакетного менеджера и состоит из статических файлов `HTML/CSS/JavaScript`.

## 3. Конфигурация

Конфигурация вынесена из кода в переменные окружения:

- `DATABASE_URL` - строка подключения к PostgreSQL;
- `AUTO_SEED` - включение тестовых данных;
- `PORT` - порт запуска API.
- `API_BASE_URL` - публичный адрес backend API для frontend-контейнера.

Пример значений находится в `.env.example`.

## 4. Сторонние службы

PostgreSQL рассматривается как внешний backing service. В локальной среде он поднимается через Docker Compose, а в облаке может быть заменен managed PostgreSQL без изменения кода, достаточно поменять `DATABASE_URL`.

## 5. Сборка, релиз, запуск

Этапы разделены:

- сборка: `docker compose build`;
- релиз: передача переменных окружения;
- запуск: `docker compose up`.

Для облака подготовлен `render.yaml`.

## 6. Процессы

API запускается как stateless-процесс `uvicorn app.main:app`. Состояние не хранится в файловой системе контейнера, бизнес-данные находятся в PostgreSQL.

## 7. Привязка портов

Backend сам слушает HTTP-порт. Порт задается через переменную `PORT`, а Docker Compose публикует его наружу как `8001:8000`.

## 8. Масштабирование

Backend не зависит от локального состояния контейнера, поэтому его можно запускать в нескольких экземплярах за балансировщиком. Общая база данных остается PostgreSQL.

## 9. Быстрый запуск и остановка

Контейнер API стартует одной командой `uvicorn`, ожидает готовность базы через healthcheck/wait logic и корректно завершается при остановке контейнера.

## 10. Паритет сред

Локальная и облачная среда используют одинаковую контейнерную модель: Python backend, PostgreSQL, статический frontend. Различаются только значения переменных окружения и адреса сервисов.

## 11. Логи

Uvicorn и приложение пишут логи в стандартный поток вывода контейнера. Их можно смотреть через:

```bash
docker compose logs api
```

В облаке эти же stdout/stderr-логи собираются платформой.

## 12. Административные процессы

Разовые административные задачи отделены от основного web-процесса. Например, наполнение тестовыми товарами можно выполнить отдельной командой:

```bash
docker compose exec api python -m app.seed_marketplace
```

Автоматический seed управляется переменной `AUTO_SEED`, поэтому в production его можно отключить.
